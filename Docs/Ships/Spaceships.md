# Spaceship

A spaceship is used by the player to travel across the galaxy. A player can own multiple ships but must either hire a pilot and crew, buy service droids, or build their own AI to run the other ships.

## Ship Properties

| Property      | Description                                                                                                                                                                                               |

| Bridge        | There are specific bridge types for each Ship Class, each with their own capabilities. This translates to features available to the player, such as different scanners, probes, and systems controls.     |

| Ship Name     | Unique name provided by a player that must be unique in the Names Registry. Every ship needs to be registered with an authority. This also gives details to the Call Sign Broadcast  ie. USS Solar Queen. |

## Ship Classifications (Ship Class)

There are several ship classifications that all models of ships fall into. These ships all contain some basic/common room type options, but certain rooms and types of equipment will only be available to ships within specific classifications.

### Exclusive Rooms and Equipment

Rooms and Equipment that require certain classifications of ship will be done based on their hardware requirements and will never explicitly state "Cruiser only". This means that users can attempt to equip inadequate ships with rooms and equipment they shouldn't normally be able to have.

> **For Example:** adding a Worm Hole Generator (normally only found on Battleships and Carriers) to a Cargo ship. It might be possible if you can create a large enough capacitor to do it, and use ALL of your cargo space for the capacitors and the WHG room. It would take a long time to charge the capacitors from the cargo engine's Power Plant, but it could be possible. Basically, if you can get enough power out of that little miner ship, you might be able to run a WHG.

## Rooms

The entire ship can be customized but a given ship will only have so much space. Player buys a ship structure that can contain multiple levels and various gridded build space for expansion. Any space not being used is automatically set as basic storage area.

Depending on the room type, a service droid or hired crew can run it autonomously. Any room that has no crew/droid must be run by the player. Various tasks can take time and have multiple steps which the player would need to manually perform. An automated room allows the player to queue up tasks that will be automatically produced. The amount of floor space in a room defines how many workers can operate in the room at the same time. Note that room equipment will take up floor space, and may reduce the available worker slots.

The player can only operate in one room at a time, which is why it is necessary for them to hire a crew that can take on tasks for them. This also means they cannot be manually piloting the ship while they are performing other actions.
